## 错误分析

1. **错误现象**：GET 请求 `/api/v1/subjects/?q=%E7%81%AB%E5%BD%B1&source=mixed&limit=20&offset=0&sort=rank` 返回 500 内部服务器错误

2. **错误类型**：类型不匹配错误

3. **问题根源**：

   * `search_subjects_endpoint` 函数中，`q` 参数是可选的 (`Optional[str]`)，默认值为 `None`

   * 当 `source=mixed` 时，函数调用 `search_mixed` 函数，传递 `q` 作为 `keyword` 参数

   * 但 `search_mixed` 函数的 `keyword` 参数定义为必需的 `str` 类型，不是 `Optional[str]`

   * 当用户不提供 `q` 参数时，`search_mixed` 函数会收到 `None`，导致类型不匹配错误

4. **错误位置**：

   * `backend/app/api/v1/subjects.py` 第 100 行：`result = await search_mixed(db, q, type, user_id, limit, offset)`

   * `backend/app/services/subject_service.py` 第 109-116 行：`search_mixed` 函数定义

## 修复方案

1. **修改** **`search_mixed`** **函数签名**：

   * 将 `keyword` 参数类型从 `str` 改为 `Optional[str]`

   * 保持函数内部逻辑不变，因为它已经处理了 `keyword` 为 `None` 的情况

2. **修复后的函数签名**：

   ```python
   async def search_mixed(
       db: AsyncSession,
       keyword: Optional[str] = None,
       subject_type: Optional[int] = None,
       user_id: Optional[int] = None,
       limit: int = 20,
       offset: int = 0
   ) -> Dict[str, Any]:
   ```

## 修复步骤

1. 打开 `backend/app/services/subject_service.py` 文件

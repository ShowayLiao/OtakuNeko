# 将后端API功能集成到FloatingActionButton组件

## 1. 功能分析

### 1.1 FAB组件现有功能
- 提供浮动操作按钮，支持展开/折叠子菜单
- 默认包含三个操作：添加收藏、导入豆瓣收藏、修改收藏
- 支持自定义操作项

### 1.2 后端API功能
根据API文档，需要集成的核心功能包括：
- 收藏管理：添加/更新收藏、获取收藏列表
- 数据同步：从BGM或豆瓣同步收藏
- 条目搜索：搜索动画、书籍等条目

## 2. 集成方案

### 2.1 API调用封装
- 引入现有`api.ts`中的API调用函数
- 为每个操作添加适当的错误处理和状态管理
- 实现请求状态显示和结果反馈

### 2.2 交互设计
- **添加收藏**：点击后打开搜索表单，用户输入关键词搜索条目，选择后添加到收藏
- **导入豆瓣收藏**：点击后打开文件选择器，用户选择豆瓣数据文件，上传并同步
- **修改收藏**：点击后打开收藏列表，用户选择要修改的收藏进行编辑

### 2.3 组件扩展
- 添加加载状态指示
- 添加成功/失败反馈提示
- 添加表单和模态框组件用于交互

## 3. 实现步骤

### 3.1 修改FloatingActionButton组件
- 引入必要的API调用函数
- 添加状态管理（加载状态、错误信息）
- 实现操作项的点击事件处理

### 3.2 添加交互组件
- 创建搜索表单组件用于添加收藏
- 创建文件上传组件用于导入豆瓣收藏
- 创建收藏列表组件用于修改收藏

### 3.3 添加错误处理
- 为每个API调用添加try-catch块
- 显示用户友好的错误信息
- 处理不同类型的错误（网络错误、服务器错误、验证错误）

### 3.4 添加状态管理
- 为每个操作添加加载状态
- 添加成功/失败反馈
- 处理API响应数据

### 3.5 测试和优化
- 测试所有API调用功能
- 优化用户交互体验
- 确保代码符合编码规范

// 工具函数

## 5. 预期效果

- 用户点击FAB按钮后，可以展开子菜单
- 点击各个操作项后，显示相应的交互界面
- 执行API调用时，显示加载状态
- 操作成功或失败后，显示相应的反馈信息
- 所有API调用都有适当的错误处理

## 6. 技术要点

- 使用React的useState和useCallback管理状态和事件处理
- 使用现有的api.ts中的API调用函数
- 添加适当的错误处理和状态管理
- 实现用户友好的交互界面
- 确保代码符合项目的编码规范和最佳实践

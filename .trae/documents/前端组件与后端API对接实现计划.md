# 前端组件与后端API对接实现计划

## 1. 项目现状分析

- 后端API文档已提供完整的接口定义、请求参数、响应格式和错误处理机制
- 前端已有基础的API服务代码(`api.ts`)和组件实现
- 部分API调用与后端文档存在差异，需要调整
- 缺少JWT认证机制的完整实现
- 部分组件的API调用需要优化和完善

## 2. 实现目标

- 确保所有前端组件的API调用与后端文档一致
- 实现完整的JWT认证机制
- 添加统一的错误处理和加载状态
- 确保数据交互的安全性和可靠性
- 验证各组件在不同场景下的正确性和稳定性

## 3. 具体实现步骤

### 3.1 API服务层优化 (`api.ts`)

1. **添加JWT认证拦截器**
   - 实现请求拦截器，自动添加Authorization头
   - 实现响应拦截器，处理token过期和认证失败

2. **调整API调用方法**
   - 根据后端文档调整现有API方法的HTTP方法、路径和参数
   - 补充缺失的API方法

3. **统一错误处理**
   - 实现统一的错误处理机制，返回标准化的错误信息
   - 添加请求超时和重试机制

4. **类型定义优化**
   - 根据后端文档完善TypeScript类型定义
   - 确保所有API请求和响应的类型安全

### 3.2 SettingsModal组件

1. **优化用户注册和登录流程**
   - 使用后端文档中的`/auth/login`接口实现用户登录/注册
   - 实现JWT token的存储和管理

2. **完善用户信息获取**
   - 使用`/users/me`接口获取当前用户信息
   - 优化用户头像上传功能

3. **添加加载状态和错误提示**
   - 为API调用添加加载状态
   - 完善错误提示信息

### 3.3 ManualAddDialog组件

1. **调整收藏更新API**
   - 将当前的`PATCH`方法改为后端文档中的`PUT`方法
   - 调整请求体格式以匹配后端要求

2. **优化搜索功能**
   - 调整搜索API的参数和响应处理
   - 添加搜索结果的缓存机制

3. **完善表单验证**
   - 加强表单验证，确保符合后端要求
   - 添加更详细的错误提示

### 3.4 GridImportModal组件

1. **调整批量导入API**
   - 确保批量导入API与后端文档一致
   - 优化导入数据的格式转换

2. **添加导入进度提示**
   - 实现更准确的导入进度显示
   - 添加导入结果的详细反馈

3. **优化搜索功能**
   - 确保搜索API与后端文档一致
   - 添加搜索结果的缓存机制

### 3.5 DoubanImportDialog组件

1. **调整豆瓣数据同步API**
   - 根据后端文档调整`/collections/sync`接口的调用
   - 确保请求体格式符合后端要求

2. **优化文件上传功能**
   - 添加文件上传的进度显示
   - 完善上传失败的错误处理

3. **添加数据验证**
   - 加强文件格式和大小的验证
   - 确保用户名符合后端要求

## 4. 测试和验证

1. **单元测试**
   - 为API服务层添加单元测试
   - 验证各API方法的正确性

2. **集成测试**
   - 测试各组件与后端API的集成
   - 验证不同场景下的功能正确性

3. **边界测试**
   - 测试空数据、无效数据、超时等边界情况
   - 验证错误处理机制的有效性

## 5. 预期成果

- 所有前端组件的API调用与后端文档完全一致
- 实现完整的JWT认证机制
- 统一的错误处理和加载状态
- 各组件在不同场景下稳定运行
- 完善的测试用例和文档

## 6. 实现顺序

1. 优化API服务层(`api.ts`)
2. 实现JWT认证机制
3. 调整各组件的API调用
4. 添加加载状态和错误提示
5. 测试和验证
6. 优化和完善

通过以上计划的实施，我们将实现前端组件与后端API的无缝对接，确保系统的稳定性和可靠性。

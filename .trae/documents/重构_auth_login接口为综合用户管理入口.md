# 重构/auth/login接口为综合用户管理入口

## 1. 改造内容

### 1.1 LoginRequest模型改造
- 修改`auth.py`中的`LoginRequest`模型，添加以下可选字段：
  - `nickname` (字符串类型，可选)
  - `bangumi_id` (整数类型，可选)
  - `avatar_url` (字符串类型，可选)
  - `sign` (字符串类型，可选)

### 1.2 响应模型改造
- 替换现有的`LoginResponse`模型，创建新的模型包含：
  - `access_token` (字符串类型)
  - `token_type` (固定值"bearer")
  - `user` (包含用户完整信息的对象)

### 1.3 登录接口逻辑重写
- 修改POST /login端点的业务逻辑：
  1. 用户身份查询：根据username查询数据库
  2. 分支业务处理：
     - 情况A（用户不存在）：创建新用户，使用请求中的可选字段（如果提供），否则使用默认值
     - 情况B（用户已存在）：选择性更新非空且变更的字段
  3. 统一令牌签发：生成7天有效期的JWT token
  4. 标准化响应：返回包含token和用户信息的完整响应

## 2. 代码改造步骤

1. 修改`auth.py`中的`LoginRequest`模型
2. 创建新的`LoginResponse`模型
3. 修改login接口的业务逻辑，实现智能判断和信息更新
4. 确保事务处理正确，使用try-except-rollback模式
5. 只更新非空且有变更的字段，避免覆盖已有数据

## 3. 预期效果

- 前端仅需提交当前表单信息，后端智能判断操作类型
- 统一返回格式，包含有效的JWT token和完整用户信息
- 支持用户首次注册和后续信息更新
- 确保数据安全性和一致性

## 4. 技术实现要点

- 使用Pydantic进行数据验证和模型定义
- 使用异步SQLAlchemy进行数据库操作
- 使用现有的`create_access_token`函数生成JWT token
- 确保username为必填字段，其他为可选字段
- 为各字段添加适当的长度限制

通过以上改造，将实现一个智能、高效的综合用户管理入口，简化前端开发，提高系统的易用性和扩展性。

# UI组件组织结构优化方案

## 1. 组件分类与职责界定

### 1.1 基础组件判断标准
- **原子性**：不可再分的最小UI单元，如按钮、输入框
- **通用性**：可在多个页面和场景中复用
- **无业务逻辑**：仅负责UI渲染，不包含业务数据处理
- **单一功能**：仅实现一个特定的UI功能

### 1.2 UI目录职责范围
- 仅包含基础UI组件
- 不包含业务逻辑组件
- 不包含布局组件
- 不包含表单组件

## 4. 组件分类体系

### 4.1 组件类型定义
- **基础UI组件**：提供基础UI元素，如按钮、输入框等
- **表单组件**：包含表单逻辑的组件，如登录表单、注册表单等
- **布局组件**：负责页面布局的组件，如页面容器、侧边栏等
- **功能组件**：实现特定功能的组件，如浮动操作按钮、主题切换器等
- **状态组件**：展示不同状态的组件，如加载状态、错误状态等
- **骨架屏组件**：展示加载过程的骨架屏
- **卡片组件**：展示卡片形式内容的组件

### 4.2 命名规范
- 组件名称使用PascalCase
- 目录名称使用kebab-case
- 组件文件名与组件名称保持一致

### 4.3 存放路径规则
- 基础UI组件：src/components/ui/
- 表单组件：src/components/forms/
- 布局组件：src/components/layout/
- 功能组件：src/components/features/
- 状态组件：src/components/state/
- 骨架屏组件：src/components/skeletons/
- 卡片组件：src/components/cards/

### 4.4 引用方式
```tsx
// 基础UI组件
import { Button } from '@/components/ui/Button';

// 表单组件
import { CollectionEditForm } from '@/components/forms/CollectionEditForm';

// 布局组件
import { PageLayout } from '@/components/layout/PageLayout';

// 功能组件
import { FloatingActionButton } from '@/components/features/FloatingActionButton';
```

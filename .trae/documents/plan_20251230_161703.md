## 错误分析

1. **错误现象**：GET 请求 `/api/v1/subjects/?q=%E7%81%AB%E5%BD%B1&source=mixed&limit=20&offset=0&sort=rank` 返回 500 内部服务器错误

2. **错误类型**：类型不匹配错误

3. **问题根源**：

   * 在上一次修复中，我们将 `search_mixed` 函数的 `keyword` 参数从 `str` 改为 `Optional[str]`

   * 但 `search_mixed` 函数会调用 `bangumi_client.py` 中的 `search_subjects` 函数

   * `bangumi_client.py` 中的 `search_subjects` 函数的 `keyword` 参数仍然定义为必需的 `str` 类型

   * 当 `search_mixed` 函数传递 `None` 作为 `keyword` 参数时，`search_subjects` 函数会收到 `None`，导致类型不匹配错误

4. **错误位置**：

   * `backend/app/services/bangumi_client.py` 第 90-96 行：`search_subjects` 函数定义

   * `backend/app/services/subject_service.py` 第 155 行：`remote_response = await search_bangumi_subjects(keyword, subject_type, limit, offset)`

## 修复方案

1. **修改** **`bangumi_client.py`** **中的** **`search_subjects`** **函数**：

   * 将 `keyword` 参数类型从 `str` 改为 `Optional[str]`

   * 添加默认值 `None`

   * 修改函数内部逻辑，当 `keyword` 为 `None` 时，返回空结果或处理空字符串

2. **修复后的函数签名**：

   ```python
   @cache(expire=86400, namespace="bangumi")
   async def search_subjects(
       keyword: Optional[str] = None,
       subject_type: Optional[int] = None,
       limit: int = 20,
       offset: int = 0
   ) -> Dict:
   ```

3. **修改函数内部逻辑**：

   * 在函数内部，确保 `keyword` 不是 `None`，例如：`keyword = keyword or ""`

   * 或者在 `keyword` 为 `None` 时返回空结果

# 数据库模型和服务实现计划

## 第一步：修改数据库模型文件 `subject.py`

### 保留的字段
* `id`：本地自增主键
* `source`：数据源

### 修改的字段
* `source_id`：映射为 `bangumi_subject.json` 中的 `id` 字段

### 完全按照 `bangumi_subject.json` 数据格式定义的字段
* `type`：条目类型
* `name`：条目原名
* `name_cn`：条目中文名
* `summary`：条目简介
* `date`：发售/放送日期
* `platform`：平台/类型
* `images`：图片字典（支持多尺寸图片）
* `image`：单个封面图片URL
* `tags`：标签列表
* `meta_tags`：官方元标签
* `infobox`：详细元数据
* `rating`：评分信息
* `collection`：收藏状态统计
* `eps`：集数
* `volumes`：卷数
* `series`：是否为系列
* `locked`：是否锁定
* `nsfw`：是否不适合儿童

### 删除的字段
* `cover_url`：这个字段在 `bangumi_subject.json` 中没有，而是使用 `image` 字段
* `score`：这个字段在 `bangumi_subject.json` 中是 `rating.score`
* `rank`：这个字段在 `bangumi_subject.json` 中是 `rating.rank`
* `collection_total`：这个字段在 `bangumi_subject.json` 中没有，而是在 `collection` 对象中
* `rating_details`：这个字段在 `bangumi_subject.json` 中是 `rating` 对象

## 第二步：更新数据访问层文件 `subject_repo.py`

### 更新 `create` 和 `save` 方法
* 修改 `source_id` 的提取逻辑，从 `subject_data.get("id")` 获取
* 修改字段映射逻辑，使用 `bangumi_subject.json` 中的字段名
* 删除对旧字段的处理
* 添加对新字段的处理

### 更新 `update` 方法
* 修改字段映射逻辑，使用 `bangumi_subject.json` 中的字段名

## 第三步：实现服务层文件 `subject_service.py` 中的CRUD基础服务

### 实现CRUD方法
* `create_subject`：创建新条目
* `get_subject_by_id`：根据ID获取条目
* `get_subject_by_source`：根据数据源和ID获取条目
* `update_subject`：更新条目
* `delete_subject`：删除条目
* `get_all_subjects`：获取所有条目

### 实现检索功能
* 本地检索：
  * `search_subject_by_id`：基于ID的精确搜索
  * `search_subject_by_name`：基于名称的宽泛搜索
* 云端检索：
  * `search_subject_cloud`：调用 Bangumi API 进行搜索
  * 集成 `bangumi_client.py` 中的 `search_subjects` 函数

### 云端检索结果处理流程
* 获取 Bangumi API 返回的 JSON 格式数据
* 将云端响应数据直接映射为本地 `Subject` 模型结构（因为字段名已经完全一致）
* 返回完整映射后的数据对象

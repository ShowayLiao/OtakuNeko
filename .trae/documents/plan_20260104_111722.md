# 整合添加/修改收藏功能计划

## 1. 组件结构设计

### 1.1 创建统一的收藏管理组件
- 整合 `SubjectSearchForm`、`CollectionAddForm` 和 `CollectionEditForm` 的功能
- 组件命名：`CollectionManager`
- 核心功能：搜索、添加收藏、修改收藏

### 1.2 组件状态管理
- `isOpen`: 控制组件显示/隐藏
- `keyword`: 搜索关键词
- `results`: 搜索结果列表
- `isSearching`: 搜索加载状态
- `isFormOpen`: 控制收藏表单显示/隐藏
- `selectedSubject`: 当前选中的条目
- `isEditing`: 区分添加/修改模式

## 2. 功能实现步骤

### 2.1 搜索功能实现
- 复用 `SubjectSearchForm` 中的搜索逻辑和 API 调用
- 实现搜索结果展示，区分已收藏和未收藏状态
- 已收藏条目显示"修改"按钮，未收藏条目显示"添加"按钮

### 2.2 统一表单设计
- 整合 `CollectionAddForm` 和 `CollectionEditForm` 的表单字段
- 设计响应式布局，包含封面展示区域和条目基本信息
- 实现表单自动回填功能：
  - 基础信息（名称、类型、封面等）从搜索结果获取
  - 收藏信息（状态、评分、评论等）仅在修改模式下从 API 获取

### 2.3 交互流程设计
1. 用户点击"添加/修改收藏"按钮
2. 显示搜索界面，用户输入关键词搜索
3. 展示搜索结果，区分已收藏/未收藏状态
4. 用户点击"添加"或"修改"按钮
5. 显示统一的收藏表单，自动回填数据
6. 用户填写/修改表单，提交保存
7. 显示操作结果反馈

## 3. 代码整合方案

### 3.1 创建 `CollectionManager.tsx` 组件
- 引入所需的 API 函数和类型
- 实现搜索、表单显示/隐藏、状态管理等核心逻辑

### 3.2 复用现有代码
- 搜索功能：复用 `SubjectSearchForm` 中的 `handleSearch` 函数
- 表单字段：复用 `CollectionAddForm` 和 `CollectionEditForm` 中的表单组件结构
- API 调用：复用现有的 `searchSubjects`、`updateOrAddCollection` 等函数

### 3.3 状态管理优化
- 使用 `useState` 和 `useCallback` 管理组件状态和回调函数
- 确保组件渲染性能优化
- 实现表单数据的正确回填和更新

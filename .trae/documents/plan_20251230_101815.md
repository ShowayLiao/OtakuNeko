# API调用方法文档生成计划

## 1. 文档结构设计
- **文档标题**：API调用方法文档
- **文档格式**：Markdown
- **文档位置**：backend根目录
- **整体结构**：
  - 文档概述
  - 认证机制说明
  - API端点分类列表
  - 每个API的详细说明（路径、方法、参数、响应、示例等）
  - 状态码说明
  - 错误处理

## 2. API端点详细分析

### 2.1 认证相关 API (/auth)
- **POST /auth/login**：用户登录/注册接口
  - 请求体：`{"username": "string"}`
  - 响应：JWT令牌
  - 认证：无需认证

### 2.2 收藏相关 API (/collections)
- **GET /collections**：获取用户收藏列表
  - 查询参数：subject_type, status, keyword, limit, offset, sort_by
  - 认证：需要JWT

- **PUT /collections/{sid}**：更新或添加收藏
  - 路径参数：sid (条目ID)
  - 请求体：收藏数据
  - 认证：需要JWT

- **POST /collections/sync**：同步收藏（支持BGM和豆瓣）
  - 请求体：source, subject_type, data
  - 认证：需要JWT

### 2.3 仪表板相关 API (/dashboard)
- **GET /dashboard/stats**：获取用户统计数据
  - 认证：需要JWT

### 2.4 条目相关 API (/subjects)
- **GET /subjects**：搜索条目
  - 查询参数：q, source, type, limit, offset, sort
  - 认证：需要JWT

- **GET /subjects/{subject_id}**：获取单个条目详情
  - 路径参数：subject_id
  - 查询参数：refresh
  - 认证：需要JWT

- **PUT /subjects/{subject_id}**：修改条目信息
  - 路径参数：subject_id
  - 请求体：更新数据
  - 认证：需要JWT

### 2.5 用户相关 API (/users)
- **GET /users/me**：获取当前用户信息
  - 认证：需要JWT

## 3. 文档生成步骤
1. 编写文档概述和认证机制说明
2. 按照API分类，逐个编写详细的端点说明
3. 为每个端点添加：
   - 功能描述
   - 请求方法和路径
   - 请求参数（路径、查询、请求体）
   - 响应格式
   - 状态码说明
   - 认证要求
   - 示例请求和响应
4. 添加全局状态码说明和错误处理部分
5. 保存文档到指定位置

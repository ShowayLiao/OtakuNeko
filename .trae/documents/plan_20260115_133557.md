# 后端系统日志配置方案

## 1. 日志配置模块设计
- **创建日志配置文件**：`app/core/logging.py`，实现完整的日志配置逻辑
- **更新配置类**：在 `app/core/config.py` 中添加日志相关配置项
- **修改主入口**：在 `app/main.py` 中初始化日志系统

## 2. 日志配置详细内容
### 2.1 日志级别
- 支持 ERROR、WARN、INFO、DEBUG 四个级别
- 通过环境变量或配置文件可调整全局日志级别

### 2.2 日志格式
- 包含时间戳、日志级别、模块名称、请求ID、具体日志内容
- 格式示例：`2026-01-15 12:00:00,000 - INFO - app.repositories.subject_repo - req-12345 - Redis cache initialized successfully`

### 2.3 日志存储
- **日志目录**：`backend/logs/`
- **文件命名**：`app-{date}.log`，如 `app-2026-01-15.log`
- **错误日志**：单独存储错误级别日志到 `error-{date}.log`

### 2.4 日志轮转
- **按日期轮转**：每天生成新的日志文件
- **按大小轮转**：单个文件达到 100MB 时自动轮转
- **保留策略**：默认保留 30 天的日志文件

### 2.5 性能优化
- **异步写入**：使用 `ConcurrentRotatingFileHandler` 或 `QueueHandler` 实现异步日志写入
- **缓冲区**：配置适当的缓冲区大小，减少 I/O 操作

### 2.6 日志清理
- **自动清理**：实现日志文件的自动清理机制
- **保留天数**：可配置的日志保留天数
- **归档机制**：超过保留期的日志文件自动归档为压缩文件

## 3. 实现步骤
1. **创建日志配置模块**：`app/core/logging.py`
2. **更新配置类**：在 `app/core/config.py` 中添加日志相关配置
3. **修改主入口**：在 `app/main.py` 中初始化日志系统
4. **创建日志目录**：确保 `backend/logs/` 目录存在
5. **更新依赖**：添加必要的依赖包（如 `concurrent-log-handler`）
6. **编写文档**：创建日志配置文档，说明日志文件路径、命名规则、轮转策略及查看方法

## 4. 技术实现要点
- 使用 Python 标准库 `logging` 模块作为基础
- 使用 `logging.handlers.TimedRotatingFileHandler` 实现按日期轮转
- 使用 `concurrent-log-handler.ConcurrentRotatingFileHandler` 实现异步写入
- 配置 `logging.Formatter` 自定义日志格式
- 实现日志上下文管理器，支持请求ID追踪
- 添加日志配置验证和错误处理机制

## 5. 预期效果
- 所有日志统一存储到指定目录
- 日志文件按日期和大小自动轮转
- 异步写入提高系统性能
- 自动清理过期日志文件
- 提供清晰的日志格式和完整的日志内容
- 支持不同级别的日志记录和筛选
